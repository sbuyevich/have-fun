# Task 06: Build First Blazor Page

## Goal

Create the first visible browser page for Stage 1.

## Work

- Build the first page as a Blazor page/component.
- Show the app name `Have-Fun`.
- Show a simple running status.
- Show the localhost URL.
- Show the LAN join URL when available.
- Add a placeholder area where Stage 2 will add name entry.
- Do not implement Game Master routing, player routing, or name validation.

## Done Criteria

- First page renders in the browser.
- Page uses Blazor `InteractiveServer` render mode.
- Page is ready for Stage 2 name input.
- Master name is not displayed publicly.

## Verification

- Run `HaveFun.Web`.
- Open the local URL.
- Confirm the page renders and shows local/LAN URL information.
